Banco Patagonia – UX/UI Redesign Case Study

Role: UX/UI Designer (Self-initiated Case Study)

Focus: Accessibility, Legacy System Modernization, Change Management.

The Challenge

Banco Patagonia’s digital platform is a high-traffic legacy system. While functionally robust, the user interface suffered from significant technical debt, presenting accessibility barriers and a fragmented navigation structure that increased cognitive load for its thousands of daily users. The primary challenge was to modernize the experience without alienating long-term users accustomed to the existing patterns, ensuring a seamless transition between the familiar and the optimized.

Strategic Research & Benchmarking

To ground the redesign in industry standards, I conducted a competitive analysis of major Argentinian and international banking platforms (including BBVA, Santander, and Banco Galicia). This benchmarking focused on three critical pillars:
  • Information Architecture: How top-tier banks categorize complex financial products.
  • Transactional Speed: Minimizing the steps required for core tasks.
  • Accessibility: Evaluating contrast ratios and touch-target sizes.
I mapped two critical user journeys—Credit Card Management and Fund Transfers—to identify where the interface was failing to guide the user efficiently

Design System & Accessibility

A foundational Design System was developed to ensure consistency across the entire ecosystem. Key decisions included:
  • UI Refresh: Modernizing the institutional color palette by adding semantic tones for success, alerts, and errors.
  • Typography & Hierarchy: Implementing a scalable typographic system aligned with WCAG standards to improve readability for elderly users and those with visual impairments.
  • Component Library: Building a custom set of responsive components designed to reduce visual noise while maintaining the platform's core structural logic

UX/UI Implementation Strategy

The redesign focuses on Progressive Disclosure—showing only the necessary information at each step to prevent user overwhelm. By improving navigation labels and simplifying the layout, I reduced the time-to-task for the most frequent operations. Recognizing the risk of "UI shock" in financial products, I developed a Phased Implementation Plan:
  • Low-Traffic Deployment: Scheduling updates to minimize disruption.
  • Omnichannel Communication: Pre-notifying users through integrated messaging.
  • Hybrid Transition: A temporary "Grace Period" where users can toggle between the legacy and the new interface to build confidence

Project Status & Vision

This project serves as a comprehensive exploration of how to handle large-scale digital transformations. It demonstrates a balance between aesthetic modernization and the pragmatic constraints of legacy banking systems, prioritizing accessibility and user retention above all
